# Nocturne Zoo / 深夜动物园

AI 规则怪谈社交游戏（Hackathon Edition, GDD v0.6 / April 2026）。

## TL;DR

- **人数**：5–10 人
- **时长**：约 6–7 分钟/局（含入场分配）
- **核心机制**：摄像头拍脸 + 2–3 道怪谈氛围选择题 → AI 分配三种动物身份（阵营）→ 玩家用“身体守则”存活并博弈 → 结算揭晓身份与判定语
- **技术栈**：PartyKit + HTML/JS + Claude API + MediaPipe + Web Audio API

## 一句话说清楚

进入游戏先对着摄像头拍一张脸，再回答 2–3 个充满怪谈氛围的问题。AI 综合你的外貌气质和回答，把你分配到三种动物之一：

- 🦁 **白狮子（守护者·净化者）**：让长颈鹿违规累计 3 次完成“净化”
- 🦉 **猫头鹰（观察者·知情者）**：结算时猜对所有其他玩家的动物身份
- 🦒 **长颈鹿（迷失者·感染者）**：让至少 1 名白狮子玩家违规死亡

每种动物有必须用身体遵守的守则（吼叫 / 不眨眼 / 甩脖子）。游戏结束时会揭晓所有人的身份与 AI 的“判定语”，争议与反差就是社交乐趣。

## 游戏流程（v0.6）

### ① 入场分配（60–90s）

- **拍照**：玩家可重拍，确认后提交
- **答题**：依次显示 2–3 道单选题
- **分析**：短暂“正在分析你…”动画
- **揭晓**：卡片翻转显示动物身份与判定语
- **私信守则卡**：给玩家推送「守则 + 胜利条件 + 同阵营成员」
- **公开广播**：仅广播“XX 已进入 🦁 区域”（不公开守则）

### ② 宣读守则（20s）

守则卡全屏展示：动物身份 + 守则全文 + 胜利条件 + 同阵营成员名单。倒计时内只能读，不能聊天。

### ③ 游戏中（4min）

实时检测身体守则执行状态。违规扣命；违规时生成怪谈叙事并广播。玩家通过聊天/观察推进阵营目标。猫头鹰暗中记录猜测。

### ④ 结算（30s）

猫头鹰提交猜测；揭示所有人身份 + 当初判定语；判定三阵营胜负。

## 三阵营与身体守则

| 动物 | 守则（检测方式） | 胜利条件 |
| --- | --- | --- |
| 🦁 白狮子 | 每 60 秒必须低吼 ≥2 秒（Web Audio） | 让至少 1 名长颈鹿违规累计 3 次（净化），白狮子全员完成才算胜利 |
| 🦉 猫头鹰 | 每 40 秒触发窗口，5 秒不能眨眼（FaceMesh） | 结算时正确猜出所有其他玩家动物身份，猫头鹰全员猜对才算胜利 |
| 🦒 长颈鹿 | 每 45 秒必须甩一次脖子（FaceMesh） | 让至少 1 名白狮子违规死亡，且长颈鹿阵营任意 1 人存活即胜 |

通用原则（GDD）：检测在客户端本地运行，不上传视频/音频；只上报违规事件给服务端。

## Claude Prompt（摘要）

入场分配建议拆 2 次调用（Vision 外貌印象先跑，答题时后台执行，减少等待）：

- **外貌分析（Vision）**：输出 2–3 句气质印象，不提具体五官/不提动物
- **综合判定**：结合外貌印象 + 题目选项，返回 JSON：`{ "animal": "...", "verdict": "..." }`
- **违规叙事**：违规时生成 2 句克制、暗示的怪谈叙事，不直说死亡

## 技术实现（当前仓库）

- `server/`：PartyKit 房间服务器 + 一个可在局域网访问的最小前端页面（用于联调）
  - `server/src/server.js`：房间状态、消息处理、广播/私信（AI 先 stub/fallback）
  - `server/public/index.html`：拍照+问卷入场流程、以及 FaceMesh 检测 demo

## 本地/局域网运行

```bash
cd server
npm install
npm run dev -- --port 0
```

启动后会打印 `127.0.0.1`（本机）和你的 **LAN IP**（同 Wi‑Fi 设备访问）。
