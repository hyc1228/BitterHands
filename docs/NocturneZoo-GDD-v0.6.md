## 概览

- **项目名**：NOCTURNE ZOO / 深夜动物园 — AI 规则怪谈社交游戏
- **玩家人数**：5–10 人
- **时长/局**：约 6–7 分钟（含入场分配阶段）
- **动物种类**：只有三种：🦁 白狮子 · 🦉 猫头鹰 · 🦒 长颈鹿
- **阵营机制**：相同动物自动同阵营，阵营胜利条件各不相同
- **入场方式**：摄像头拍脸 + 2–3 个问题 → AI 综合判断分配动物身份
- **核心乐趣**：被 AI 看穿「你像哪种动物」· 用身体守则存活 · 三方阵营博弈
- **技术栈**：PartyKit + HTML/JS + Claude API + MediaPipe + Web Audio API

## 1. 一句话说清楚游戏

进入游戏先对着摄像头拍一张脸，再回答 2–3 个充满怪谈氛围的问题。AI 综合你的外貌气质和回答，判断你更像哪种动物，把你分配到对应的阵营。三个阵营各有截然不同的目标：白狮子要「净化」长颈鹿，猫头鹰要「观察」并猜出所有人的身份，长颈鹿要「扩散感染」让白狮子死亡。每种动物有必须用身体遵守的守则——狮子要定时吼叫，猫头鹰五秒不能眨眼，长颈鹿要定时甩脖子。三方同时博弈，游戏结束才揭晓所有人的身份、判定语和最终胜负。

核心爽点：AI 凭什么说我像长颈鹿？——这个问号本身就是游戏开始前最好的社交话题。结算时所有人的判定语一起揭晓，往往比胜负更好笑。

## 2. 入场分配机制

这是 v0.6 相对 v0.5 最核心的变化：用「AI 看脸 + 问卷」代替「画画识别」，让分配过程本身变成一个社交仪式——每个人都想知道 AI 凭什么把自己判定成那种动物。

### 2.1 入场四步流程

1) **拍照（10s）**：摄像头拍一张正脸照片（玩家可以重拍，满意后确认提交）  
AI：Claude Vision 分析面部特征：轮廓、眼神、表情状态，生成一段简短的「外貌印象」文字（不展示给玩家，只用于后续综合判断）

2) **答题（30–45s）**：依次显示 2–3 个选择题，玩家点击作答。题目有怪谈氛围，选项之间没有明显「正确答案」  
AI：记录玩家的选择，结合外貌印象，构建一份「soul profile」——玩家的气质标签组合

3) **分析（5s）**：屏幕短暂黑屏，出现「正在分析你…」动画  
AI：Claude 综合外貌 + 回答，从三种动物中选出最匹配的一种，并生成一句「判定语」

4) **揭晓（10s）**：动物身份以戏剧化方式揭晓（卡片翻转动画），展示判定语。所有人的分配结果依次广播给全体  
AI：私密推送守则卡片给该玩家（含动物守则 + 胜利条件 + 同阵营成员）

体验设计意图：从拍照到揭晓控制在 60–90 秒。等待时看到「正在分析你…」的动画，揭晓时的卡片翻转动画要有仪式感。判定语是整个入场体验的高光时刻，哪怕结果本身不准，玩家也会觉得「有点道理」或「完全离谱但好笑」。

### 2.2 问题设计

问题总数 3 道，每道都是单选，选项没有明显「对错」，只是气质倾向不同。问题以动物园怪谈世界观包裹，让玩家感觉自己在做一份「真实的档案」。

| 问题 | 选项 A | 选项 B | 选项 C | 倾向 |
| --- | --- | --- | --- | --- |
| 深夜，你一个人走在动物园里，发现园区的灯全灭了。你的第一反应是？ | 站在原地，等眼睛适应黑暗，观察四周 | 大声喊出来，告诉任何可能在附近的人你在这里 | 继续往前走，感觉黑暗没什么好怕的 | 🦉 / 🦁 / 🦒 |
| 你捡到一张纸条，上面写着「不要相信你看到的大象」。你会怎么做？ | 把纸条收好，悄悄记在心里，继续观察 | 立刻告诉所有人，让大家一起警惕 | 看了看大象，觉得没问题，把纸条扔掉 | 🦉 / 🦁 / 🦒 |
| 你发现同行的人眼神开始变得奇怪，你认为他们不再是原来的人了。你选择？ | 什么都不说，继续观察，等待更多证据 | 直接质问他们，你是谁 | 默默靠近他们，感觉和他们在一起更安全 | 🦉 / 🦁 / 🦒 |

问题设计原则：A 选项通常倾向猫头鹰（静默观察）；B 选项倾向白狮子（主动发声）；C 选项倾向长颈鹿（随流漂移）。但 Claude 的判定会结合外貌综合判断，所以「全选 A」不一定变猫头鹰。

### 2.3 Soul Profile：AI 如何综合判断

Claude 的判断依据来自两个维度——外貌气质（Vision 分析）和问题回答（文字分析），综合成一份内部的「soul profile」，再映射到三种动物。

判定不追求「准确」，追求「有趣」。AI 可以判定错，玩家会觉得「完全不准」也是好的游戏体验——因为他们会去讨论「我明明应该是猫头鹰」。争议本身就是社交价值。

### 2.4 判定语示例

| 动物 | 判定语示例 |
| --- | --- |
| 🦁 白狮子 | 「你的眼神里有一种不容置疑的东西。动物园需要你。」 / 「你不是来参观的。你从一开始就知道这里需要守护者。」 |
| 🦉 猫头鹰 | 「你看起来像一个已经在观察我们很久了的人。你属于深夜。」 / 「你的眼睛告诉我，你从来不错过任何细节。动物园选择了你作为它的目击者。」 |
| 🦒 长颈鹿 | 「有什么东西已经开始改变你了。你自己也感觉到了，对吗？」 / 「你的气质里有一种漂移感。动物园认出了你——你不是第一次来这里了。」 |

### 2.5 Claude Prompt 设计

入场分配涉及两次 Claude 调用，加上游戏中的违规叙事调用，共三种 prompt 场景：

1) **外貌分析（Vision）**（纯后端）  
System: 你是深夜动物园的「感知者」。你能从一个人的外貌判断他们在这个动物园里属于哪种存在。  
User: [图片] 用2-3句话描述这个人给你的直觉印象。不要提到具体五官，只描述气质和感觉。不要提动物名称。

2) **综合判定 + 判定语**（纯文字）  
System: 你是深夜动物园的「感知者」，只识别三种存在：白狮子（守护者·主动·发声）、猫头鹰（观察者·静默·知情）、长颈鹿（迷失者·漂移·被感染）。  
User: 外貌印象：[①的输出]  
问题回答：Q1选了[A/B/C]，Q2选了[A/B/C]，Q3选了[A/B/C]  
请判断此人属于哪种动物，并生成一句判定语（20字以内，怪谈风格，不要直接说动物名称）。  
只回复 JSON：`{"animal": "白狮子", "verdict": "判定语文本"}`

3) **违规叙事**（游戏中）  
System: 你是深夜动物园规则怪谈的叙事引擎。  
User: [玩家名]是一只[动物]，它刚才[违规描述]。用2句话描述随之发生的诡异现象，语气克制、充满暗示，不要直接说死亡。

关键设计：两次调用分开（外貌分析先做，拿到结果后再结合答案判定），这样可以让 Vision 分析在玩家答题时就在后台跑，减少等待时间。判定结果用 JSON 格式返回，方便解析。

## 3. 三阵营设计

| 阵营 | 设定 | 守则 | 胜利条件 | 信息 |
| --- | --- | --- | --- | --- |
| 🦁 白狮子 | 守护者·净化者 | 每隔 60 秒必须发出一次持续 2 秒以上的低吼声（麦克风检测） | 让至少 1 名长颈鹿玩家违规累计 3 次（净化）。白狮子全员完成才算阵营胜利。 | 知道：自己守则 + 同阵营成员；不知道：猫头鹰是谁 |
| 🦉 猫头鹰 | 观察者·知情者 | 每 40 秒触发一次检测窗口，窗口内持续 5 秒不能眨眼（FaceMesh） | 结算时正确猜出所有其他玩家的动物身份。猫头鹰全员猜对才算胜利。 | 知道：所有人的动物类型；不知道：其他猫头鹰的存在 |
| 🦒 长颈鹿 | 迷失者·感染者 | 每隔 45 秒必须做一次甩脖子动作（FaceMesh） | 让至少 1 名白狮子玩家违规死亡。长颈鹿任意 1 人存活即算胜利。 | 知道：自己守则 + 同阵营成员；不知道：猫头鹰是谁 |

分配平衡：AI 判定不强制平衡阵营人数。不均衡的分配本身就是游戏变量。

## 5. 身体守则检测

通用原则：

- 所有检测在客户端本地运行，不传视频/音频数据给服务端
- 只上报违规事件：`{ type: 'violation', animal: 'owl', player: 'xxx' }`
- 每种检测都有容错缓冲，避免误判
- 入场拍照时已申请摄像头权限，守则卡片展示时申请麦克风权限

## 6. 猫头鹰的猜测机制

猫头鹰开局知道所有人的动物类型（服务端分配完成后私密推送完整名单），但不能直接说出来；结算时提交猜测列表。

## 7. 阵营互动关系

- 🦁 vs 🦒：核心对抗（净化 vs 扩散）
- 🦉 vs 所有人：信息博弈
- 🦁 vs 🦉：暗含背叛/伪装
- 🦒 vs 🦉：混淆与反制

## 10. 技术实现

模块与方案：

- 多人实时：PartyKit（房间管理、状态广播、私密消息）
- 前端：单页 HTML + Vanilla JS（不需要构建工具）
- 入场拍照：`getUserMedia` + `canvas.toDataURL()` base64
- AI 外貌分析：Claude Vision（claude-haiku）
- AI 综合判定：Claude（claude-haiku）输出 JSON `{animal, verdict}`
- 眨眼检测：MediaPipe FaceMesh（本地）
- 头部移动检测：MediaPipe FaceMesh（本地）
- 声音检测：Web Audio API
- AI 叙事：Claude Haiku（违规时服务端调用并广播）

服务端消息处理（摘要）：

- `join`：记录玩家，等待房主开始
- `submit_photo`：base64 → Vision 外貌印象 → 暂存
- `submit_answers`：结合外貌印象判定动物 → 私信守则卡 → 广播入场
- `start`：广播开局，启动计时器
- `violation`：扣命 → AI 叙事 → 广播 → 检查净化条件
- `chat`：转发全体
- `owl_submit`：猫头鹰提交猜测
- `end`：计时结束 → 判定胜负 → 广播结算（含判定语）
